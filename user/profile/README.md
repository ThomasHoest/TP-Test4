# Profile — who you are

The durable picture of you that the assistant reasons from, built up through
conversation and onboarding. Kept as a few small, focused files rather than one
big one.

## What's here

- `about.md` — identity: name, location, how you'd like to be addressed.
- `work.md` — your role, company, and current work context.
- `working-style.md` — how you like to work.
- `working-with-me.md` — how the assistant should help you (your preferences).
- `uploads/` — source documents you've dropped (e.g. a CV), extracted to text.

## When to write here vs. elsewhere

- Who you are / how you work → here.
- What you're aiming at → [`../goals/`](../goals/).
- Your recurring cadence → [`../routines/`](../routines/).
- Working drafts → [`../tasks/`](../tasks/); notes about other people → [`../people/`](../people/).
