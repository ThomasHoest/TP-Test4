# Projects — long-running efforts

One file per long-running piece of work — a strategy presentation, a
campaign, a hiring round. Naming: `<slug>.md`.

A project is bigger and multi-step, and may spawn several tasks. Distinct
from [`../tasks/`](../tasks/), which holds small, discrete to-dos (e.g.
"write a LinkedIn post").

## When to write here vs. elsewhere

- A larger, multi-step or ongoing effort → a project file here.
- A single quick to-do → [`../tasks/`](../tasks/).
- A decision you've reached → [`../decisions/`](../decisions/).
