# Memory taxonomy

This table is the routing registry for your memory. It says, for each kind of
information, **where it lives** and **what shape** the file has. The memory map
(`index.md`, regenerated by the system every turn) is a projection of this
table over the repo — so **editing this table evolves the structure**, with no
code change. Add a row to introduce a new category; the next turn's map picks it
up. Files that carry a `type:` the table doesn't list show up under
"Unregistered types" in the map as candidates to add here.

| Category      | Destination                                   | Shape            |
|---------------|-----------------------------------------------|------------------|
| identity      | user/profile/about.md                         | sections         |
| work          | user/profile/work.md                          | sections         |
| working-style | user/profile/working-style.md                 | sections         |
| how-to-help   | user/profile/working-with-me.md               | sections         |
| goal          | user/goals/goals.md                           | sections         |
| routine       | user/routines/routines.md                     | sections         |
| person        | user/people/<slug>.md                         | per-entity + fm  |
| project       | user/projects/<slug>.md                       | per-entity + fm  |
| task          | user/tasks/<YYYY-MM-DD>-<slug>.md             | per-entity       |
| decision      | user/decisions/decisions.md                   | append + date    |
| observation   | agent/memory/observations.md                  | append + conf.   |
| log           | agent/memory/log/<YYYY>/<MM>/<YYYY-MM-DD>.md  | per-entity + fm  |
